## 에셋 워크플로우 (sync-icons 스크립트)

타워 아이콘 PNG 교체는 **`npm run sync-icons`** 한 줄로 끝남.

1. 새 PNG를 `src/image/`에 **게임의 타워 키 그대로** 저장:
   - `sniper.png`, `cannon.png`, `frost.png`, `tesla.png`, `laser.png`,
     `frostgun.png`, `fireworks.png`, `inferno.png`, `tornado.png`, `mechanest.png`
2. `npm run sync-icons` 실행 → `public/assets/icons/`로 자동 복사
   (md5 일치하면 스킵, 누락된 파일은 보고)
3. `src/scenes/TitleScene.ts`의 `ICON_FILES`에서 해당 타워가 `.png`로 설정됐는지 확인
4. 브라우저 하드 리프레시 (`Ctrl+Shift+R`)

매핑 테이블은 `scripts/sync-icons.mjs` 상단의 `MAPPING` 객체. 현재 컨벤션은
**1:1** (`sniper.png` → `sniper.png`) — 소스/대상 동일 이름.

PNG 권장 사양: 256×256 정사각, 투명 배경, 라벨/텍스트 없음. 출시 전 TinyPNG
등으로 압축 (현재 1~2MB → 200KB 이하 목표).

## 관찰된 이슈 / 개선 후보

### 밸런스 / 게임플레이
- [ ] 카드 풀 16장 → 특정 카드 등장 빈도 ~19%. 가중치 도입 검토 (유틸 카드 빈도 ↑, 강력 타워 빈도 ↓)
- [ ] Frost (data.damage=1) 너무 약하지 않은지 — 슬로우 효과만 의미 있음
- [ ] Power Up 카드 중첩 시 후반 너무 강해짐 (1.25^3 ≈ 2배). 캡 검토
- [ ] Repair 카드 (+5) 과대 — 베이스 max 15 대비 1/3 회복. 적정 +3 ?
- [ ] Wave 5 → Boss 격차 검토 — 5웨이브 클리어 후 보스가 너무 쉽거나 어려운지
- [ ] 보스 데미지 5 — 한 마리 누수 시 즉사 위험. 적정 검토

### UX / UI
- [ ] **타워 제거 불가** — 한번 놓으면 영구. 비싼 카드 잘못 놓으면 재시작 외 답 없음
- [ ] **Undo 환불 없음** — 모양 블럭 잘못 놓으면 토큰 손해
- [ ] **격자 경계 도달 시 확장 불가** — 격자 8×12 한도에서 막힘. 메시지 표시?
- [ ] **첫 플레이 튜토리얼 없음** — 토큰 → 노란 셀 → 탭 흐름 처음 보면 헷갈림
- [ ] 카드 풀에서 같은 타워 카드 반복 등장 시 정보 동일 (인풋 다양성 ↓)
- [ ] 화면 작은 폰에서 카드 모달 가독성 (현재 320px 높이 카드, 작은 폰에서 잘림 가능)

### 시각 자산
- [ ] 4 타워 PNG 미생성 (Tesla / Frost Gun / Fireworks / Tornado) — SVG 폴백 중
- [ ] 적 (Grunt / Runner / Tank / Boss) 그림 없음 — 컬러 원 + 단일 글자
- [ ] 베이스 그림 없음 — 컬러 사각형
- [ ] 길/스폰/베이스 셀 시각 단조 (단색 사각형)
- [ ] PNG 비정사각형 시 stretch 가능 (현재 setDisplaySize 강제 정사각) — 종횡비 유지로 변경 검토

### 사운드 (전체 미진행)
- [ ] BGM (메뉴 / 매치 / 보스)
- [ ] SFX (사격 / 처치 / 카드 픽 / UI 탭 / 베이스 피격 / 매치 결과)

### 코드 품질 / 기술 부채
- [ ] `drawIcon` 종횡비 미보존 (`setDisplaySize`로 강제 정사각)

- [ ] Phaser 4 마이그레이션 변동 (예: `lineBetween` 호환성) — 향후 패치 시 점검

## 디자인 결정 — 이미 결정된 사안

> 이미 합의되어 변경하지 않을 결정. 새로운 의문이 생기면 여기 먼저 확인.

- ✅ 스폰 확장 모델 — BFS 자유 배치 폐기. 단방향(역방향) 자동 회전.
- ✅ 길 블럭 영구 — 한번 놓으면 제거 불가 (Undo 없음, 환불 없음)
- ✅ 타워 영구 — 한번 놓으면 제거/이동 불가
- ✅ 시작 카드 풀 = 타워/모양만 — 첫 픽에 즉시 쓸 수 있는 것만
- ✅ 베이스 자동 방어 = 반경 2칸 = 140px = 2 × cellSize 70
- ✅ 모양 1 카드 = 1 토큰 = 1 배치 (다중 셀) — I/L/U 모두 동일 패턴
- ✅ 카드 풀 16장 — 픽당 3장 무작위 추출 (가중치 미적용)
