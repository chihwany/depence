# 04. Post-MVP 구현 로드맵

> MVP(6주차) 완료 이후의 큰 그림. 각 Phase는 **이전 Phase 완료를 전제**로 한다.
> 각 Phase 끝에는 **검증 게이트**(계속 갈지 멈출지 결정 포인트)를 둘 것 — 직선으로 다 진행하지 말 것.

---

## 전체 흐름

```
MVP (6주) ─→ Phase 1 (깊이) ─→ Phase 2 (메타) ─→ Phase 3 (인게이지먼트)
                                                        ↓
Phase 7 (운영) ←─ Phase 6 (수익화) ←─ Phase 5 (폴리시) ←─ Phase 4 (백엔드)
                                                        ↓
                                              [Soft Launch — 약 6~7개월차]
```

---

## Phase 1: Game Depth (게임 깊이) — 4~6주

**목표**: MVP 코어 루프를 "한 판 더 하고 싶은" 깊이로 확장

| 항목 | 작업 |
|------|------|
| **지형 빌드 시스템** | 격자 기반 길/타워 배치 (Raid Rush 스타일) |
| 타워 확장 | 3종 → 8~12종 (각자 명확한 특화) |
| 적 확장 | 4종 → 8~10종 (저항/면역 같은 속성 도입) |
| 카드 풀 확장 | 6종 → 20~30종 |
| 맵 추가 | 1종 → 5~8종 (각 맵 고유 격자/지형) |
| 챕터별 난이도 곡선 | 적 스케일링 공식 |

---

## Phase 2: Meta Progression (메타 진행) — 4~6주

**목표**: 매치 사이의 성장으로 장기 플레이 동기 형성

| 항목 | 작업 |
|------|------|
| 영구 화폐 | 골드(소프트), 잼(프리미엄) |
| 카드 컬렉션 | 매치 후 영구 카드 획득 |
| 카드 업그레이드 | 중복 카드 + 골드 → 등급업 (Common→Rare→Epic→Legendary) |
| 플레이어 레벨 | 매치 XP, 레벨업 보상 |
| 배틀 덱 구성 | 매치 입장 전 8장 덱 선택 |
| 챕터/스테이지 | 클리어 별 ★ 보상 |

---

## Phase 3: Engagement Systems (인게이지먼트) — 3~4주

**목표**: 일일/주간 접속 동기 형성

| 항목 | 작업 |
|------|------|
| 데일리 로그인 | 7일 사이클, 점증 보상 |
| 데일리 미션 | "오늘 3매치 승리" 류 |
| 위클리 미션 | "이번 주 보스 5회 처치" 류 |
| 업적 | 일회성 도전 (~50개) |
| 푸시 알림 | 복귀 유도 |

---

## Phase 4: Backend / Accounts (백엔드) — 3~4주

**목표**: 다기기 동기화, 랭킹, 사회 기능 기반

| 항목 | 작업 |
|------|------|
| 인증 | Firebase Auth (Apple/Google 로그인) |
| 클라우드 저장 | Firestore 진행도 동기화 |
| 리더보드 | 베스트 웨이브, 클리어 시간 |
| 프로필 | 닉네임, 아바타, 통계 |
| (선택) 친구 시스템 | 친구 추가, 소셜 매칭 |

---

## Phase 5: Production Polish (프로덕션 폴리시) — 3~4주

**목표**: "출시 가능한 수준"의 시청각 품질

| 항목 | 작업 |
|------|------|
| 정식 아트 | 타워/적 스프라이트, UI 일러스트 (외주 또는 직접) |
| 애니메이션 | 사격/죽음/피격 (Spine 또는 프레임) |
| 파티클 | 폭발, 슬로우, 환경 |
| BGM | 메뉴/매치/보스 (3~5곡) |
| SFX | UI/사격/처치/카드 픽 (~20종) |
| 로컬라이제이션 | 한국어 + 영어 (확장: 일본어/중국어) |
| 성능 최적화 | 60fps 안정 (저사양 기기 기준) |

---

## Phase 6: Monetization (수익화) — 2~3주

**목표**: 출시 가능한 수익 구조

| 항목 | 작업 |
|------|------|
| Rewarded Ads | AdMob via Capacitor — 부활, 보상 박스, 더블 리워드 |
| IAP | RevenueCat — 잼 패키지, 스타터 팩, NoAds |
| 배틀패스 | 시즌(4~8주) 단위, Free/Premium 트랙 |
| 한정 번들 | 신규 유저 패키지, 위크엔드 세일 |
| 분석 | Firebase Analytics + GameAnalytics 통합 |

---

## Phase 7: Launch & Live Ops (출시 + 운영) — Ongoing

**목표**: 출시 후 살아있는 게임 유지

| 항목 | 작업 |
|------|------|
| Soft Launch | 1~2개 국가 (캐나다, 필리핀 등) → 지표 확인 |
| Global Launch | 전세계 출시 |
| 시즌 운영 | 4~8주 단위 신규 컨텐츠 |
| 라이브 이벤트 | 한정 카드, 콜라보, 스킨 |
| 패치 | 월 1~2회 밸런스/버그 |
| CS | 유저 문의/리뷰 대응 |

## 일정 요약 (대략)

```
주차      Phase                          상태
-----     -----------------------------  -----------
1~6       MVP                            진행 중
7~12      Phase 1: Game Depth            ← 다음
13~18     Phase 2: Meta
19~22     Phase 3: Engagement
23~26     Phase 4: Backend
27~30     Phase 5: Polish
31~33     Phase 6: Monetization
34        Soft Launch
36+       Global + Live Ops              [무기한]
```

→ **MVP 끝나고 출시까지 약 6~7개월** (풀타임 1인 기준)

---

## 솔직하게 짚을 점

위 일정은 **풀타임 1인 기준 빡빡한 베스트 케이스**다. 현실은:

| 형태 | 출시까지 |
|------|---------|
| 풀타임 1인 | 6~9개월 (위 일정 +30~50%) |
| 파트타임 1인 (저녁/주말) | 12~18개월 |
| 2~3인 팀 | 4~6개월 |

게다가 **출시 ≠ 성공**. 인디는 보통 5~10개 출시 후 한 개 히트가 평균이다. 한 게임에 12개월+ 투자하는 결정은 매우 신중해야 한다.

---

## 단계별 검증 게이트

전체를 직선으로 가지 말 것. 각 Phase 끝에서 **다음 Phase로 갈지 / 피벗할지 / 중단할지** 결정한다.

| 게이트 | 검증 질문 | 통과 못 하면 |
|--------|----------|-------------|
| **MVP 끝** | 친구 5~10명에게 시켰을 때 "한 판 더?" 나오나? | 코어 피벗 |
| **Phase 1 끝** | 30~50명 테스트 시 D7 리텐션 ≥ 10%? | 게임 컨셉 재고 |
| **Phase 2~3 끝** | 100명 테스트 시 D7 ≥ 15% / 평균 세션 5분+? | 메타 시스템 재설계 |
| **Phase 4~5 끝** | Soft Launch 직전 인터널 테스트 통과? | 폴리시 더 |
| **Soft Launch** | LTV / CPI > 1? | 수익화 / 마케팅 재설계 |

각 게이트에서 통과 못 하면 다음 Phase로 안 가는 게 좋다. 안 그러면 6~12개월 다 투자한 뒤에야 결과를 알게 된다.

---

## 의사결정 원칙

1. **데이터 없이 다음 Phase로 넘어가지 말 것** — 매번 실유저 테스트 후 결정
2. **Phase 순서는 권장이지 강제가 아님** — 데이터에 따라 순서 바꿔도 됨
3. **출시 = 시작**. 출시 후 6개월이 진짜 게임의 운명을 결정 — 운영 여력을 항상 확보할 것
4. **포기할 용기**. 검증 게이트에서 막히면 미련 없이 피벗 또는 다음 프로젝트로 넘어갈 것
