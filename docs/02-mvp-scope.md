# 02. MVP 범위 정의

> 4~6주 안에 만들 최소 기능셋. 목표는 **"재미가 있는지 검증"**이지 "출시"가 아니다.

---

## MVP 한 줄 정의

> **한 맵, 3 타워, 3 적, 5 웨이브, 카드 픽 1회/웨이브 — 한 판이 재밌는지 확인한다.**

---

## 포함 (IN SCOPE)

### 게임 플레이
- [ ] 1개 맵 (직선 또는 단순 곡선 경로)
- [ ] 타워 배치 슬롯 6~8개 (고정 위치)
- [ ] 5웨이브 + 1보스 웨이브
- [ ] 3종 타워 (저격형 / AOE / 슬로우)
- [ ] 3종 적 (일반 / 빠름 / 탱크)
- [ ] 베이스 HP 시스템 (적 도달 시 감소)
- [ ] 매치 승리/패배 판정

### 카드 픽 시스템
- [ ] 웨이브 클리어 시 3장 중 1장 선택
- [ ] 카드 효과: 새 타워 배치권 / 기존 타워 +1렙 / 단순 패시브
- [ ] 인매치 한정 (매치 종료 시 초기화)

### UI
- [ ] 타이틀 화면 → "Play" 버튼
- [ ] 인게임 HUD (베이스 HP, 웨이브 번호, 다음 웨이브 카운트다운)
- [ ] 카드 선택 모달
- [ ] 결과 화면 (Win/Lose + Retry)

### 시스템
- [ ] 로컬 저장 (최고 기록만, `localStorage`)
- [ ] 30/60 fps 안정 동작 (모바일 브라우저 기준)
- [ ] 720x1280 세로 화면 + 반응형

## 제외 (OUT OF SCOPE — 의도적으로 안 만든다)

| 빠진 것 | 이유 |
|---------|------|
| 메타 진행 (영구 카드 수집) | 코어 검증이 우선 |
| 다중 맵 | 1맵으로 검증 가능 |
| 백엔드 / 계정 | Firebase는 코어 검증 후 |
| 광고 / IAP | 출시 단계의 일 |
| 리더보드 / 멀티플레이 | MVP 범위 폭증 위험 |
| 사운드 / BGM | 늦게 추가해도 됨 |
| 정식 그래픽 에셋 | placeholder(컬러 도형)로 진행 |
| 스프라이트 애니메이션 | Phaser 트윈으로만 처리 |
| 다국어 | 한국어 하드코딩 |
| 모바일 빌드 (Capacitor) | 6주차에 한 번 |

---

## 6주 일정

| 주차 | 목표 | Done 기준 |
|------|------|-----------|
| 1주 ✓ | 환경 셋업 | `npm run dev` 동작 확인 |
| 2주 | 코어 루프 (1 타워, 1 적, 1 웨이브) | 적이 걸어오고 타워가 쏘고 죽음 |
| 3주 | 카드 픽 + 타워/적 종류 확장 | 3종 타워, 3종 적, 5웨이브 |
| 4주 | UI + 매치 흐름 | 타이틀 → 플레이 → 결과 → 재시작 |
| 5주 | 밸런싱 + 본인 플레이 테스트 | "한 판이 재미있다" 자체 평가 |
| 6주 | Capacitor 빌드 + 실기기 확인 | 본인 폰에서 동작 |

## "재미 있다"의 판단 기준

5주차 자체 평가 시 다음 3개 질문에 모두 **YES**면 통과:

1. **한 판이 끝났을 때 "한 판 더" 누르고 싶은가?**
2. **카드 픽에서 고민이 되는가?** (모든 카드가 명백히 좋거나 나쁘면 X — 의사결정이 없는 것)
3. **다른 빌드(다른 카드 조합)로 시도해보고 싶은가?**

→ 3개 다 YES면 다음 단계(메타 시스템) 진행.
→ 하나라도 NO면 코어 튜닝 더 (밸런싱, 카드 효과 재설계).

## 밸런싱 수식 시작 템플릿

본인이 플레이하면서 튜닝 — 아래는 **시작점일 뿐, 정답이 아님**.

### 적 HP / 스폰 곡선

```
적 HP (웨이브 N)    = base_hp * (growth_hp ^ (N-1))
적 스폰 수 (웨이브 N) = base_count + spawn_step * (N-1)

예시 (튜닝 시작값):
  base_hp = 10, growth_hp = 1.25
  → 1~5웨이브: 10, 12.5, 15.6, 19.5, 24.4

  base_count = 8, spawn_step = 2
  → 1~5웨이브: 8, 10, 12, 14, 16
```

### 타워 DPS 곡선

```
타워 DPS (레벨 L) = base_dps * (growth_dps ^ (L-1))

예시:
  base_dps = 5, growth_dps = 1.5
  → 1~5렙: 5, 7.5, 11.3, 16.9, 25.3
```

### 황금 비율 — 성장 곡선의 차이

```
적 HP 성장률(1.25) < 타워 DPS 성장률(1.5)
```

→ 타워 강화가 적 강화보다 빨라야 **"성장하는 느낌"**이 난다.
→ 다만 너무 차이 크면 게임이 너무 쉬워짐. 1.25 vs 1.5는 시작점이고,
   플레이테스트로 "후반에 빡빡하지만 클리어 가능"한 지점을 찾는다.

### 경제 (소프트 화폐)

```
적 처치 보상 = 적_난이도 * 1
웨이브 클리어 보너스 = wave_N * 10
타워 배치 비용 = 타워별 고정 (예: 100, 150, 80)
타워 업그레이드 비용 = base_cost * (1.5 ^ current_level)
```

→ 매 웨이브 끝났을 때 **"다음 타워 1개 추가 또는 업그레이드 1회"** 가능한 정도가 적정.
→ 너무 부족하면 답답, 너무 풍부하면 의사결정 없음.
